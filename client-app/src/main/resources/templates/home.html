Flujo implementado

Authorization Code Flow

El login ocurre en el IdP. La app cliente solo recibe tokens después de que el usuario supera contraseña y segundo factor TOTP.

  • 1
    La app redirige al IdP mediante /oauth/authorize.
  • 2
    El IdP valida correo, contraseña y código TOTP.
  • 3
    El IdP devuelve un authorization code al /callback.
  • 4
    La app intercambia el code por access token y refresh token.
  • 5
    La app valida el JWT usando el JWKS público del IdP.
  • 6
    El refresh token puede rotarse para renovar el access token.
