Capacidades implementadas

Flujo de identidad completo

La aplicación cliente no accede a la clave privada del IdP. Valida tokens mediante la clave pública expuesta en JWKS.

  • ✓
    Registro con contraseña hasheada usando BCrypt.
  • ✓
    Verificación de correo mediante SMTP real.
  • ✓
    Login con segundo factor TOTP basado en tiempo.
  • ✓
    OAuth Authorization Code Flow simplificado.
  • ✓
    JWT firmado con RS256 y clave privada protegida.
  • ✓
    Refresh token con rotación, revocación e introspección.
  • ✓
    Auditoría de eventos críticos de autenticación.
