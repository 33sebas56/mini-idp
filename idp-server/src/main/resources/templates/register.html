Registro en el servidor de identidad

Cree una cuenta local en el IdP. Después del registro se enviará un correo de verificación mediante SMTP real.

La contraseña se almacena usando BCrypt. El correo debe verificarse antes de iniciar sesión.

Seguridad del registro

Validación inicial de identidad

El registro no habilita el acceso inmediatamente. Primero se valida el correo, luego el usuario debe iniciar sesión y configurar TOTP.

  • ✓
    Contraseña protegida con hash BCrypt.
  • ✓
    Correo enviado por SMTP real.
  • ✓
    Token de verificación con expiración.
  • ✓
    El login se bloquea si el correo no está verificado.
  • ✓
    El acceso completo requiere TOTP.
